Integrar agentes de IA com ferramentas como Slack, email e webhooks é o passo que transforma uma análise isolada em um fluxo de trabalho completo e automatizado. Com a Sofia AI, você conecta o resultado de qualquer orquestração diretamente ao canal certo — sem precisar de código adicional ou plataformas intermediárias.

## Por que integrar saídas de IA com ferramentas externas?

Agentes de IA produzem valor real quando o resultado chega para a pessoa certa, no momento certo, no lugar certo. Imagine um agente que analisa concorrentes toda manhã: se o relatório fica preso em um painel que ninguém abre, o valor se perde. A integração resolve isso.

Os três canais mais usados por times de produto, marketing e operações são:

- **Slack**: notificações em tempo real no canal do time
- **Email**: relatórios formatados enviados automaticamente para stakeholders
- **Webhooks HTTP**: disparo de eventos para qualquer sistema externo (CRM, ERP, banco de dados)

## Integração com Slack via Incoming Webhooks

O Slack oferece uma funcionalidade chamada Incoming Webhooks, que gera uma URL única para cada canal. Ao fazer um `POST` com um payload JSON nessa URL, uma mensagem aparece automaticamente no canal configurado.

### Configurando o Incoming Webhook no Slack

1. Acesse [api.slack.com/apps](https://api.slack.com/apps) e crie um novo app
2. Ative a funcionalidade **Incoming Webhooks** no menu lateral
3. Clique em **Add New Webhook to Workspace** e escolha o canal de destino
4. Copie a URL gerada — ela tem o formato `https://hooks.slack.com/services/XXX/YYY/ZZZ`

### Formato da mensagem

```json
{
  "text": "Análise de concorrentes concluída!",
  "blocks": [
    {
      "type": "section",
      "text": {
        "type": "mrkdwn",
        "text": "*Relatório de Concorrentes — 24/02/2026*\n\nO agente identificou 3 mudanças de preço e 2 novos produtos lançados."
      }
    }
  ]
}
```

### Como configurar no Sofia AI

Na Sofia AI, ao criar ou editar uma orquestração, você encontra a seção **Integrações de Saída**. Basta colar a URL do Incoming Webhook do Slack e definir o template da mensagem usando as variáveis de saída do agente. Sem código, sem intermediários.

## Integração com Email (Resend e SendGrid)

Para notificações por email, os serviços mais usados por times técnicos são o **Resend** e o **SendGrid**. Ambos oferecem APIs simples e alta entregabilidade.

### Quando usar email em vez de Slack?

| Situação | Canal recomendado |
|---|---|
| Notificação imediata para o time | Slack |
| Relatório semanal para diretores | Email |
| Alerta de erro crítico | Slack + Email |
| Resumo executivo mensal | Email |

### Exemplo de payload para o Resend

```javascript
const response = await fetch('https://api.resend.com/emails', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer SUA_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    from: 'sofia@suaempresa.com.br',
    to: 'diretoria@suaempresa.com.br',
    subject: 'Relatório de Concorrentes - ' + new Date().toLocaleDateString('pt-BR'),
    html: '<h1>Análise concluída</h1><p>{{resultado_orquestracao}}</p>'
  })
})
```

Na Sofia AI, você configura o provedor de email uma única vez nas configurações do workspace e depois seleciona o destinatário em cada orquestração.

## Integração com Webhooks HTTP Genéricos

Webhooks HTTP são a forma mais flexível de integrar IA com qualquer sistema. Ao final de uma orquestração, a Sofia AI faz um `POST` para a URL configurada, enviando o resultado no formato JSON.

### Casos de uso de webhooks

- **CRM**: ao analisar um lead com IA, enviar o score automaticamente para o Salesforce ou HubSpot
- **Banco de dados**: armazenar o resultado de uma análise em uma tabela do Supabase ou Postgres
- **Zapier/Make**: disparar um fluxo de automação maior a partir do resultado da IA
- **Sistema interno**: atualizar o status de um processo no ERP da empresa

### Estrutura do payload enviado pela Sofia AI

```json
{
  "orchestration_id": "orch_abc123",
  "orchestration_name": "Análise de Concorrentes",
  "status": "completed",
  "completed_at": "2026-02-24T09:30:00Z",
  "output": {
    "summary": "Foram identificadas 3 mudanças relevantes...",
    "raw_data": {}
  },
  "metadata": {
    "tokens_used": 4200,
    "duration_seconds": 47
  }
}
```

### Configurando um webhook no Sofia AI

1. Acesse a orquestração desejada no painel
2. Clique em **Configurações de Saída**
3. Selecione **Webhook HTTP**
4. Cole a URL do seu endpoint
5. Opcionalmente, adicione headers de autenticação (ex: `Authorization: Bearer seu_token`)

## Caso prático: Notificação automática de análise de concorrentes

Veja como um time de produto usa o Sofia AI para se manter atualizado sobre a concorrência sem nenhuma intervenção manual.

### O fluxo completo

1. **Agendamento**: a orquestração "Análise de Concorrentes" roda automaticamente todo dia às 8h
2. **Execução**: os agentes coletam dados de 5 concorrentes, comparam preços, funcionalidades e menções em redes sociais
3. **Saída Slack**: ao concluir, uma mensagem é enviada ao canal `#inteligencia-competitiva` com o resumo executivo
4. **Saída Email**: o relatório completo em HTML é enviado para o time de diretoria
5. **Saída Webhook**: o resultado em JSON é enviado para o CRM para atualizar os registros de análise

### Resultado para o time

- Zero tempo gasto em coleta manual de dados
- Diretor recebe o relatório antes da reunião matinal
- CRM sempre atualizado com a inteligência de mercado mais recente
- Time foca em decisão estratégica, não em coleta de informação

## Boas práticas de integração

### Segurança nos webhooks

- Sempre use HTTPS — nunca HTTP — em URLs de webhook
- Configure um token secreto para validar que a requisição vem da Sofia AI
- Limite o acesso ao endpoint de webhook apenas para IPs conhecidos, quando possível

### Confiabilidade nas notificações

- Configure uma notificação de fallback: se o Slack falhar, o email garante a entrega
- Monitore o histórico de execuções no painel da Sofia AI para identificar falhas de entrega
- Use canais de erro dedicados no Slack para alertas de falha

### Formatação das mensagens

- Mensagens de Slack devem ser concisas — use o email para relatórios longos
- Inclua um link direto para o resultado completo na plataforma
- Use emojis no Slack para indicar status (ex: ✅ concluído, ⚠️ atenção, ❌ erro)
