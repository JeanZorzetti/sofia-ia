## O que é um Workflow de IA

Um workflow de IA é uma sequência de etapas automatizadas onde agentes de inteligência artificial executam tarefas específicas — lendo, analisando, escrevendo ou tomando decisões com base em dados.

**Diferente de automação tradicional**, workflows de IA não apenas movem dados entre sistemas. Eles entendem, raciocinam e geram outputs criativos.

**Exemplos:**
- Receber um email de cliente → entender o problema → gerar resposta personalizada → encaminhar para o setor correto
- Receber currículo → analisar qualificações → comparar com requisitos da vaga → gerar relatório de fit
- Receber transcrição de reunião → extrair action items → enviar resumo por email para participantes

## Os Blocos Fundamentais de um Workflow de IA

Antes de construir, entenda os blocos:

### 1. Trigger (Gatilho)
O que inicia o workflow. Pode ser:
- Mensagem recebida (WhatsApp, email, chat)
- Upload de arquivo
- Horário programado (cron)
- Webhook de sistema externo
- Chamada manual

### 2. Agente(s) de IA
O(s) "cérebro(s)" do workflow. Cada agente tem:
- **System prompt** — define personalidade, regras e objetivo
- **Knowledge Base** — documentos de referência (FAQ, políticas, manuais)
- **Modelo** — qual LLM usar (Llama 3.3, Claude, GPT-4)

### 3. Conectores
Pontes para sistemas externos:
- CRM (HubSpot, Pipedrive)
- Email (Gmail, Outlook)
- WhatsApp (Evolution API)
- Banco de dados
- Planilhas

### 4. Saída (Output)
O que o workflow produz:
- Texto/resposta enviada ao usuário
- Arquivo gerado (PDF, CSV)
- Dado salvo em sistema
- Notificação enviada

## Passo a Passo: Criando seu Primeiro Workflow

Vamos criar um workflow completo: **Qualificação automática de leads via WhatsApp**.

### Cenário
Alguém manda mensagem no WhatsApp perguntando sobre seu serviço. O agente qualifica, responde com informações relevantes e avisa o time de vendas apenas se for um lead qualificado.

---

### Etapa 1: Defina o objetivo com clareza

Antes de abrir qualquer ferramenta, responda por escrito:

- **Input:** Mensagem de texto livre de prospect no WhatsApp
- **Processo:** Identificar interesse, qualificar fit, responder com informação adequada
- **Output:** Resposta ao prospect + notificação interna se qualificado
- **Critérios de qualificação:** Empresa com mais de 10 funcionários, segmento relevante, budget percebido

---

### Etapa 2: Construa a Knowledge Base

No Sofia AI, vá em **Knowledge Base → Nova Base**:

1. Nomeie: "Qualificação de Leads"
2. Faça upload de:
   - Perguntas frequentes sobre seu produto (FAQ.pdf)
   - Tabela de preços (Pricing.pdf)
   - Casos de uso por segmento (Cases.pdf)
3. Aguarde o processamento (3-5 minutos)

Pronto. Seu agente agora "sabe" tudo sobre seu produto.

---

### Etapa 3: Configure o Agente

Vá em **Agentes → Novo Agente**:

**Nome:** Qualificador de Leads

**System Prompt:**
```
Você é um especialista em vendas consultivas da [Empresa].

Seu objetivo: entender o problema do potencial cliente e verificar se
nossa solução é adequada para ele.

Processo:
1. Cumprimente de forma calorosa e profissional
2. Faça no máximo 2 perguntas para entender o contexto (empresa, tamanho, problema atual)
3. Com base nas respostas, verifique na Knowledge Base se temos caso de uso relevante
4. Se for lead qualificado (empresa 10+ pessoas, segmento alvo): apresente solução + proponha demo
5. Se não for qualificado: seja honesto, redirecione para recursos gratuitos

Não minta sobre preços. Se não tiver a informação, diga que vai verificar.
Tom: consultivo, sem pressão de venda, genuinamente útil.
```

**Knowledge Base:** Selecione "Qualificação de Leads"

**Modelo:** llama-3.3-70b-versatile (bom equilíbrio custo/qualidade)

---

### Etapa 4: Conecte ao WhatsApp

Em **Integrações → WhatsApp**:

1. Insira as credenciais da sua instância Evolution API
2. Selecione o agente "Qualificador de Leads"
3. Configure o horário de resposta automática (ex: 8h-22h)
4. Ative

Agora toda mensagem recebida no WhatsApp passa pelo seu agente.

---

### Etapa 5: Configure a notificação de lead qualificado

No system prompt, você pode instruir o agente a adicionar uma tag interna quando detectar lead qualificado. Combine isso com um webhook para notificar o time via Slack ou email.

---

### Etapa 6: Teste antes de ativar

Use o modo **Sandbox** do Sofia AI:
- Simule 5 conversas diferentes (lead qualificado, não qualificado, pergunta técnica, reclamação, spam)
- Verifique se as respostas estão corretas
- Ajuste o system prompt onde necessário

## Dicas para Workflows de Alta Performance

### Seja específico no system prompt
Prompts vagos geram respostas medianas. Quanto mais contexto, regras e exemplos você der, melhor a performance.

**Evite:** "Seja útil e responda perguntas dos clientes"

**Prefira:** "Você é o Carlos, especialista de suporte da [Empresa]. Seu objetivo é resolver dúvidas sobre [produto específico]. Se não souber a resposta, diga: 'Vou verificar com a equipe e retorno em até 2 horas.'"

### Itere com dados reais
Após 50-100 execuções, analise onde o agente errou. Atualize o prompt. Repita.

### Não tente fazer tudo em um agente
Agentes sobrecarregados com múltiplas responsabilidades perdem foco. Separe responsabilidades quando a complexidade aumentar.

### Monitore o que importa
- Taxa de resolução autônoma (sem escalar para humano)
- Satisfação do usuário (NPS/CSAT)
- Tempo médio de resposta
- Custo por execução
