**Orquestração de agentes IA** é a coordenação de múltiplos agentes de inteligência artificial para resolver tarefas complexas de forma colaborativa, onde cada agente executa uma função específica e passa o resultado para o próximo.

Se um único agente IA é como um funcionário generalista, uma orquestração é como um **time especializado** trabalhando em paralelo — cada um focado no que faz melhor.

---

## Por que orquestrar múltiplos agentes?

Um único agente IA tem limitações práticas:

- **Contexto limitado:** Modelos têm janela de contexto finita. Um agente que tenta fazer pesquisa + análise + redação ao mesmo tempo perde qualidade em cada etapa.
- **Sem especialização:** "Fazer tudo" geralmente significa "fazer tudo mediano".
- **Difícil de debugar:** Quando o resultado é ruim, não sabe onde falhou.
- **Inflexível:** Trocar o modelo ou a estratégia exige reescrever tudo.

Com orquestração:

| Benefício | Explicação |
|---|---|
| **Especialização** | Cada agente faz uma coisa muito bem |
| **Paralelismo** | Agentes podem executar em paralelo |
| **Modularidade** | Troque um agente sem quebrar os outros |
| **Rastreabilidade** | Veja exatamente onde o processo falhou |
| **Escalabilidade** | Adicione etapas sem reescrever tudo |

## Como funciona uma orquestração de agentes IA

### Arquitetura básica: pipeline sequencial

```
Input → [Agente A] → [Agente B] → [Agente C] → Output
```

A saída de cada agente se torna o input do próximo. É o modelo mais simples e mais comum.

**Exemplo — Pipeline de conteúdo:**
```
Brief do cliente
    ↓
[Pesquisador] — busca referências e dados relevantes
    ↓
[Copywriter] — escreve o artigo com base na pesquisa
    ↓
[Revisor SEO] — otimiza para palavras-chave e legibilidade
    ↓
Artigo publicável
```

### Arquitetura avançada: orquestrador + especialistas

Um agente "maestro" delega tarefas para agentes especializados conforme necessário:

```
[Orquestrador]
    ├── [Especialista A] — pesquisa
    ├── [Especialista B] — análise de dados
    └── [Especialista C] — redação
```

O orquestrador decide qual especialista chamar, em que ordem e com que inputs.

## 5 exemplos de orquestrações prontas para usar

### 1. Pipeline de marketing de conteúdo

**Agentes:** Pesquisador → Copywriter → Revisor → Formatador SEO

**Input:** Tema ou palavra-chave alvo

**Output:** Artigo completo, otimizado para SEO, com meta description

**Tempo:** 3-5 minutos (vs. 3-4 horas humanas)

---

### 2. Pipeline de atendimento ao cliente multinível

**Agentes:** Triagem → Especialista de Produto → Especialista Financeiro → Escalação

**Input:** Mensagem do cliente

**Output:** Resposta resolvida ou escalada para o humano certo com contexto completo

**Diferencial:** Clientes nunca repetem o problema para diferentes atendentes

---

### 3. Pipeline de qualificação de leads

**Agentes:** Coletor → Pesquisador de Empresa → Qualificador → Gerador de Abordagem

**Input:** Email ou LinkedIn do lead

**Output:** Score de qualificação + email personalizado de abordagem

**Métricas típicas:** Taxa de resposta 3x maior vs. email genérico

---

### 4. Pipeline de análise jurídica

**Agentes:** Extrator → Analisador de Cláusulas → Pesquisador de Jurisprudência → Redator de Parecer

**Input:** Contrato em PDF

**Output:** Parecer jurídico com riscos identificados e recomendações

**Tempo:** 15 minutos (vs. 4-8 horas de advogado júnior)

---

### 5. Pipeline de recrutamento

**Agentes:** Triagem → Qualificador Técnico → Analisador Cultural → Gerador de Ranking

**Input:** Vagas de currículo em PDF/texto

**Output:** Ranking dos candidatos com justificativa para cada posição

Em `/dashboard/orchestrations`, clique em "Nova Orquestração".

### Passo 2: Adicione os agentes

### Passo 3: Conecte os agentes

Arraste uma linha do output do Agente A para o input do Agente B. A saída de um automaticamente se torna o input do próximo.

### Passo 4: Configure o trigger

Defina o que dispara a orquestração:
- Mensagem de WhatsApp
- Formulário do site
- Chamada de API
- Agendamento (cron)

O Sofia AI tem um modo de execução em tempo real onde você vê cada agente processando, com logs detalhados de tokens usados, tempo e custo.

## Erros comuns em orquestrações

**❌ Agentes muito genéricos**
Se o prompt do agente é vago ("Você é um assistente útil"), o resultado será mediano. Cada agente deve ter uma especialidade clara e limitada.

**❌ Passar contexto demais entre agentes**
Não envie o histórico completo de cada agente para o próximo. Passe apenas o output relevante. Isso reduz custo e melhora a qualidade.

**❌ Não ter fallback de erro**
Se um agente falha, o pipeline todo trava. Configure comportamentos de fallback: "se o Pesquisador falhar, o Copywriter usa o conhecimento que já tem".

**❌ Orquestrar o que não precisa ser orquestrado**
Nem toda tarefa precisa de múltiplos agentes. Se um agente resolve sozinho, use um agente. Orquestre quando a complexidade justifica.

## Perguntas frequentes

**Orquestração multi-agente é difícil de configurar?**
Com Sofia AI, não. O editor visual permite criar uma orquestração completa em menos de 1 hora, sem código. Existem também 3 templates prontos para começar.

**Qual a diferença entre orquestração e automação com N8N?**
N8N automatiza fluxos de dados entre sistemas (quando X acontece, faça Y). Orquestração de agentes envolve raciocínio e linguagem natural — os agentes tomam decisões, não só movem dados.

**Posso misturar modelos diferentes em uma orquestração?**
Sim. No Sofia AI, cada agente pode usar um modelo diferente. Você pode usar Groq (rápido e barato) para triagem e GPT-4o para a tarefa mais crítica.

**Quanto custa executar uma orquestração?**
Depende dos modelos e volume de tokens. Com Sofia AI e modelos Groq, uma orquestração típica custa centavos por execução. O painel de analytics mostra o custo exato por execução.
