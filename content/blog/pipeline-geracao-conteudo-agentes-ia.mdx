Criar conteúdo de qualidade de forma consistente é um dos maiores gargalos de times de marketing e produto. Em 2026, a resposta mais eficiente para esse problema não é "usar o ChatGPT para escrever artigos" — é construir um **pipeline de agentes IA especializados**, onde cada agente tem uma função específica e passam o trabalho adiante.

Este guia mostra como construir esse pipeline do zero usando o Sofia AI.

---

## Por que pipeline de agentes, não um único LLM?

A tentação comum é pedir para um único modelo "escreva um artigo completo de blog sobre X". O resultado costuma ser:
- Tom genérico, sem personalidade
- Estrutura previsível (intro → 3 pontos → conclusão)
- SEO fraco (o modelo não sabe quais keywords priorizar)
- Sem verificação de fatos
- Conteúdo que parece... escrito por IA

O problema não é o modelo — é a abordagem. Um único agente generalista tentando fazer pesquisa + escrita + SEO + revisão ao mesmo tempo fica na mediocridade em todas as dimensões.

**A solução: separação de responsabilidades.** Cada agente é especialista em uma coisa só.

---

## A arquitetura do pipeline

```
[Tema/Keyword]
       ↓
[Agente Pesquisador]
  - Pesquisa a intenção de busca
  - Levanta os principais pontos a cobrir
  - Identifica ângulos únicos
       ↓
[Agente Copywriter]
  - Escreve o rascunho completo
  - Tom de voz consistente com a marca
  - Estrutura narrativa clara
       ↓
[Agente SEO Reviewer]
  - Verifica uso de keywords
  - Sugere title tag e meta description
  - Valida heading hierarchy
       ↓
[Agente Editor]
  - Revisa fluidez e clareza
  - Remove clichês e frases genéricas
  - Formata para publicação
       ↓
[Conteúdo publicável]
```

Cada passo tem um input bem definido (a saída do anterior) e um output bem definido (o que passa para o próximo).

## Configurando no Sofia AI

### Passo 1: Criar a orquestração

Acesse `/dashboard/orchestrations` e crie uma nova orquestração com estratégia **Sequencial**.

Dê um nome como "Pipeline de Blog" e configure a entrada como um campo de texto onde você vai inserir o tema/keyword.

### Passo 2: Agente Pesquisador

**System Prompt:**
```
Você é um especialista em pesquisa de conteúdo para marketing digital.

Dado um tema/keyword, você deve:
1. Identificar a intenção de busca principal (informacional, comercial, transacional)
2. Listar os 5-7 subtópicos mais importantes que um artigo sobre esse tema deve cobrir
3. Identificar 2-3 ângulos únicos ou perspectivas diferenciadas
4. Listar as 3-5 keywords secundárias relacionadas
5. Descrever o perfil do leitor ideal

Formato de saída: JSON estruturado com campos: intencao, subtopicos, angulos_unicos, keywords_secundarias, perfil_leitor.
```

**Por que JSON:** o próximo agente precisa de dados estruturados, não texto livre.

### Passo 3: Agente Copywriter

**System Prompt:**
```
Você é um copywriter especializado em conteúdo técnico B2B para SaaS.

Tom de voz: direto, prático, sem floreios. Evite clichês como "no dinâmico mundo de", "em constante evolução", "cada vez mais". Escreva como uma pessoa real explicando para outra pessoa real.

Você receberá um JSON com pesquisa sobre o tema. Use essas informações para:
1. Escrever um artigo completo em Markdown com mínimo de 1200 palavras
2. Incluir exemplos de código quando relevante
3. Usar a estrutura: Introdução → Seções H2 → Conclusão com CTA
4. Mencionar o Sofia AI naturalmente no contexto (não forçado)

Escreva apenas o conteúdo do artigo em Markdown, sem frontmatter.
```

### Passo 4: Agente SEO Reviewer

**System Prompt:**
```
Você é um especialista em SEO técnico para conteúdo de blog.

Você receberá o rascunho de um artigo em Markdown. Sua tarefa é:
1. Propor um title tag otimizado (50-60 caracteres, keyword no início)
2. Propor uma meta description (140-155 caracteres, com call-to-action)
3. Verificar se a keyword principal aparece no H1, nos primeiros 100 palavras, e em pelo menos 2 H2s
4. Listar sugestões de internal links (para /features/orchestrations, /templates, /blog)
5. Pontuar a densidade de keyword (ideal: 1-2%)

Formato de saída: o artigo original com suas sugestões de title/description no topo como comentários Markdown, e anotações [SEO: ...] inline onde necessário.
```

### Passo 5: Agente Editor

**System Prompt:**
```
Você é um editor de conteúdo experiente. Seu trabalho é refinar o texto final para publicação.

Instruções:
1. Remova qualquer anotação [SEO: ...] do agente anterior, incorporando as sugestões silenciosamente
2. Verifique a fluidez de leitura — parágrafos longos devem ser quebrados
3. Garanta que cada seção tem uma transição clara para a próxima
4. Verifique se o artigo termina com um CTA claro para o Sofia AI
5. Retorne o artigo final em Markdown limpo, pronto para publicação

Não reescreva o conteúdo — apenas refine e formate.
```

## Executando o pipeline

Com a orquestração configurada, a execução é simples:

1. Insira o tema: `"Como configurar Google OAuth no Next.js 16"`
2. Clique em "Executar"
3. Acompanhe o streaming em tempo real de cada agente
4. Copie o output final do Agente Editor

O processo completo leva entre 2-5 minutos dependendo do comprimento do artigo.

## Variações do pipeline

### Para redes sociais (mais rápido)

```
[Tema]
  ↓ Agente Copywriter (com prompt para thread/carrossel)
  ↓ Agente Adaptador (versões para LinkedIn, Twitter, Instagram)
```

### Para email marketing

```
[Tema + Segmento]
  ↓ Agente Pesquisador
  ↓ Agente Redator de Email (tom conversacional, sem markdown)
  ↓ Agente de Subject Line (5 variações para teste A/B)
```

### Para SEO em escala (múltiplos artigos)

Use a API do Sofia AI para enviar múltiplos temas em paralelo e processar o output programaticamente.

## Adicionando Knowledge Base ao pipeline

Para conteúdo que precisa de informações específicas da sua empresa (casos de uso, dados, produto), vincule uma Knowledge Base ao Agente Pesquisador:

1. Faça upload de documentos relevantes (whitepapers, dados de produto, cases)
2. No editor de orquestração, vincule a KB ao Agente Pesquisador
3. O agente agora pesquisa tanto seu prompt quanto sua KB

Isso garante que o conteúdo gerado reflita a realidade do seu produto, não generalizações do modelo.

---

## Publicação automática

Para fechar o loop completamente, configure um webhook de output na orquestração:

- **Webhook URL**: endpoint da sua plataforma de CMS (Notion, Contentful, etc.)
- **Email**: envie o rascunho para revisão humana antes de publicar
- **Slack**: notifique o time de marketing quando um novo artigo estiver pronto

Assim, o pipeline vai do tema ao rascunho pronto-para-revisão sem intervenção manual.
