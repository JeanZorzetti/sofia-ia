Criar uma orquestração do zero leva tempo — você precisa definir cada agente, escrever os prompts, montar o pipeline e testar o fluxo. Mas e se você pudesse começar com um template já configurado, testado e pronto para usar?

O Sofia IA oferece 5 templates de orquestração pré-configurados para os casos de uso mais comuns em empresas brasileiras. Um clique para criar, alguns minutos para customizar, e você tem uma automação funcional em produção.

---

## Por que usar templates prontos

Templates de orquestração são mais do que atalhos — são boas práticas cristalizadas. Cada template foi desenhado com:

- **Divisão clara de papéis**: Cada agente tem uma função específica e não tenta fazer tudo
- **Prompts otimizados**: Os prompts de sistema foram refinados para cada função
- **Fluxo de dados testado**: A ordem dos agentes e como os resultados são passados foi validada
- **Instruções em português**: Todos os prompts funcionam com conteúdo em PT-BR

Você pode usar os templates diretamente ou como ponto de partida para customização.

## Template 1: Equipe de Marketing

**Categoria**: Criação de Conteúdo
**Tempo estimado**: 45-90 segundos
**Modelo recomendado**: Groq Llama-3.3 ou GPT-4o

### O que faz

Este pipeline transforma um briefing ou tema em conteúdo completo e revisado. Funciona para artigos de blog, newsletters, posts para LinkedIn, scripts de vídeo e qualquer formato textual.

### Agentes envolvidos

**1. Pesquisador**
- Analisa o tema fornecido
- Identifica os ângulos mais interessantes e relevantes para o público-alvo
- Busca dados, estatísticas e exemplos que embasam o conteúdo
- Estrutura os pontos principais que devem ser cobertos
- (Com KB): Consulta documentos internos da empresa para incluir informações exclusivas

**2. Copywriter**
- Recebe o briefing do Pesquisador
- Escreve o conteúdo completo com introdução, desenvolvimento e conclusão
- Adapta o tom para o formato solicitado (formal para whitepaper, casual para redes sociais)
- Inclui CTAs e elementos de persuasão quando apropriado

**3. Revisor**
- Lê o texto produzido pelo Copywriter
- Verifica clareza, coesão, gramática e ortografia
- Melhora a estrutura de parágrafos que estejam confusos
- Garante que o tom está consistente do início ao fim
- Entrega a versão final pronta para publicação

### Resultado esperado

Um texto completo, revisado e pronto para publicação. Para artigos de blog, tipicamente entre 600-1.200 palavras bem estruturadas. Para posts de LinkedIn, 150-300 palavras com boa cadência de leitura.

### Como ativar

1. Vá em **Orquestrações** no Sofia IA
2. Clique em **Usar Template**
3. Selecione "Pipeline de Marketing"
4. Customize os prompts se quiser adaptar o tom ou formato
5. Execute com seu briefing como input

## Template 2: Suporte Inteligente

**Categoria**: Atendimento ao Cliente
**Tempo estimado**: 20-40 segundos
**Modelo recomendado**: Groq Llama-3.3 (velocidade para atendimento)

### O que faz

Automatiza o primeiro nível de atendimento ao cliente. O pipeline recebe a mensagem do cliente, classifica o tipo de problema, elabora a resposta com base na Knowledge Base da empresa e decide se precisa escalar para um humano.

### Agentes envolvidos

**1. Triagem**
- Lê a mensagem do cliente
- Classifica o tipo de solicitação: dúvida técnica, reclamação, pedido de reembolso, bug, feature request, elogio
- Extrai informações chave: nome do produto, versão, urgência, contexto
- Determina o nível de prioridade (P1 urgente, P2 normal, P3 baixo)
- (Com KB): Verifica se é uma pergunta frequente com resposta conhecida

**2. Atendente Especialista**
- Recebe a triagem do primeiro agente
- (Com KB): Busca na base de conhecimento a informação mais relevante para o problema
- Elabora uma resposta completa, empática e técnica quando necessário
- Inclui os passos para resolução quando for um problema técnico
- Escreve em tom adequado ao contexto (mais formal para reclamações, mais amigável para dúvidas simples)

**3. Agente de Escalação**
- Avalia se a resposta elaborada é adequada
- Verifica se há informações faltando que só um humano poderia fornecer
- Decide: "Resposta automática aprovada" ou "Escalar para humano"
- Se escalar, inclui um resumo do contexto para o atendente humano

### Resultado esperado

Para 70-80% dos casos, uma resposta completa pronta para enviar ao cliente. Para os 20-30% restantes, um resumo claro do problema com o contexto necessário para o atendente humano resolver rapidamente.

### Como ativar

Recomendamos conectar uma Knowledge Base com a documentação de produto antes de ativar. O template funciona sem KB, mas a qualidade das respostas técnicas melhora muito com documentação de suporte.

## Template 3: Pesquisa e Análise

**Categoria**: Business Intelligence e Pesquisa
**Tempo estimado**: 60-120 segundos
**Modelo recomendado**: GPT-4o (para análises mais profundas)

### O que faz

Transforma uma pergunta ou tema em um relatório analítico completo. Ideal para análise de mercado, pesquisa competitiva, síntese de tendências e relatórios executivos.

### Agentes envolvidos

**1. Coletor**
- Recebe o tema ou pergunta de pesquisa
- Estrutura os dados coletados de forma organizada
- Identifica lacunas de informação
- (Com KB): Busca dados proprietários da empresa que são relevantes para a análise
- Entrega um relatório de dados brutos bem estruturado

**2. Analista**
- Recebe os dados coletados pelo Coletor
- Identifica padrões, tendências e correlações
- Separa o que é fato do que é interpretação
- Mapeia oportunidades e riscos
- Constrói a análise em seções: Contexto, Fatos Relevantes, Insights, Implicações

**3. Sintetizador**
- Recebe a análise completa do Analista
- Cria uma versão executiva: os pontos mais importantes em linguagem acessível
- Adiciona recomendações práticas baseadas na análise
- Formata o output para ser compartilhado diretamente com stakeholders

### Resultado esperado

Um relatório completo em três camadas: dados brutos (do Coletor), análise aprofundada (do Analista) e síntese executiva com recomendações (do Sintetizador). O output do Sintetizador é o que vai para o relatório final.

### Como ativar

Este template é especialmente poderoso quando combinado com uma KB contendo relatórios anteriores, dados internos e benchmarks do setor. O Coletor vai cruzar automaticamente os dados externos com as informações proprietárias.

## Template 4: Assessoria Jurídica

**Categoria**: Jurídico e Compliance
**Tempo estimado**: 60-90 segundos
**Modelo recomendado**: GPT-4o (para precisão jurídica)

### O que faz

Auxilia na análise de documentos jurídicos, identificação de riscos em contratos e elaboração de pareceres iniciais. Não substitui um advogado, mas acelera muito o trabalho de triagem e análise.

### Agentes envolvidos

**1. Pesquisador Jurídico**
- Lê o documento ou questão jurídica fornecida
- Identifica os pontos críticos: cláusulas de risco, obrigações, prazos, penalidades
- (Com KB): Busca em contratos anteriores e jurisprudência interna do escritório
- Faz um mapeamento estruturado dos pontos que precisam de análise

**2. Analista Jurídico**
- Recebe o mapeamento do Pesquisador
- Avalia cada ponto crítico em profundidade
- Identifica riscos legais, cláusulas abusivas, inconsistências
- Compara com padrões de mercado e melhores práticas
- Classifica cada risco por severidade (crítico, moderado, baixo)

**3. Redator de Parecer**
- Recebe a análise completa do Analista
- Transforma em um parecer formal bem estruturado
- Inclui sumário executivo, análise detalhada e recomendações
- Escreve em linguagem jurídica apropriada mas acessível ao cliente
- Indica quais cláusulas precisam de negociação ou modificação

### Resultado esperado

Um parecer inicial completo com identificação de riscos, análise de cláusulas e recomendações de ação. Ideal para triagem rápida de contratos antes da revisão completa por um advogado.

### Como ativar

Crie uma KB com contratos modelo, pareceres anteriores e qualquer jurisprudência relevante para o escritório ou empresa. Quanto mais completa a KB, mais preciso o Pesquisador Jurídico será ao identificar precedentes relevantes.

## Template 5: Business Intelligence Paralelo

**Categoria**: BI e Analytics
**Tempo estimado**: 45-90 segundos (paralelo = mais rápido)
**Modelo recomendado**: Groq Llama-3.3 para coletores + GPT-4o para analista

### O que faz

Usa orquestração **paralela** para coletar dados de múltiplas perspectivas simultaneamente e depois consolida tudo em um dashboard analítico. Ideal para monitoramento de mercado, análise competitiva em tempo real e relatórios multi-dimensional.

### Agentes envolvidos

**Coletores Paralelos (executam ao mesmo tempo)**

- **Coletor A — Perspectiva de Mercado**: Analisa o contexto macro do mercado, tendências de setor, movimentos de demanda
- **Coletor B — Perspectiva Competitiva**: Analisa os concorrentes, posicionamento, pricing, movimentos recentes
- **Coletor C — Perspectiva Interna** (com KB): Analisa os dados internos da empresa — performance, conversão, feedback de clientes

Como são paralelos, os três executam ao mesmo tempo. O tempo total é o do mais lento, não a soma dos três.

**Analista Consolidador (executa após os coletores)**

- Recebe os outputs dos três coletores simultaneamente
- Identifica padrões cross-perspectiva: onde os dados internos se alinham ou divergem do mercado
- Destaca oportunidades que aparecem na intersecção das três perspectivas
- Identifica riscos que se revelam quando você combina os dados

**Agente de Dashboard**

- Recebe a análise consolidada
- Formata em um relatório executivo com seções claras
- Cria um sumário "Top 5 ações recomendadas"
- Inclui métricas-chave e indicadores para acompanhamento

### Resultado esperado

Um relatório BI completo e multi-perspectiva em menos tempo do que levaria para fazer sequencialmente. A combinação de perspectivas paralelas frequentemente revela insights que análises single-track perdem.

### Como ativar

Este template brilha quando o Coletor C tem acesso a uma KB com dados internos da empresa. Conecte planilhas de performance, relatórios de vendas e feedback de clientes à KB para que a perspectiva interna seja realmente relevante.

## Como personalizar os templates

Os templates são pontos de partida, não receitas fixas. Você pode:

**Adicionar agentes**: Se precisar de mais uma camada de especialização, adicione um agente no meio do pipeline. Por exemplo, no template de Marketing, você pode adicionar um **Agente SEO** entre o Copywriter e o Revisor.

**Ajustar os prompts**: Cada agente tem um prompt de sistema que pode ser editado. Se o Copywriter está escrevendo em tom muito formal para sua marca, ajuste para um tom mais casual.

**Trocar os modelos**: Agentes que precisam de velocidade (triagem, coleta) podem usar Groq. Agentes que precisam de qualidade máxima (análise jurídica, síntese executiva) podem usar GPT-4o.

**Conectar KBs**: Cada agente pode ter sua própria KB ou compartilhar uma. Um agente de atendimento pode ter a KB de suporte; um agente jurídico, a KB de contratos.

**Combinar templates**: Você pode criar orquestrações que usam partes de templates diferentes. Por exemplo, usar o Pesquisador do template de Marketing + o Sintetizador do template de Pesquisa.
